# Telegram-бот для апскейла и улучшения фото через Replicate API

## Задача

Создать Telegram-бота на Python (aiogram 3), который принимает фото от пользователя,
последовательно применяет два шага обработки через Replicate API:
1. Апскейл 4x (Real-ESRGAN) — увеличение разрешения
2. Улучшение качества (face/detail enhance) — детали, освещение, кожа

Затем возвращает итоговое улучшенное фото пользователю.

Окружение: Python 3.11+, pip.

---

## Стек

- `aiogram` 3.x — асинхронный Telegram-бот
- `replicate` — официальный Python SDK для Replicate API
- `aiohttp` — скачивание фото из Telegram и результатов от Replicate
- `python-dotenv` — переменные окружения
- `logging` + `RotatingFileHandler` — лог в `bot.log` (макс. 5 МБ, 3 файла)

## Сценарий работы

```
1. Пользователь отправляет фото
         ↓
2. Бот отвечает: "⏳ Шаг 1/2: Увеличиваю разрешение (4x)..."
         ↓
3. Replicate: Real-ESRGAN — апскейл 4x
         ↓
4. Бот обновляет статус: "⏳ Шаг 2/2: Улучшаю качество..."
         ↓
5. Replicate: clarity-upscaler или tencentarc/gfpgan — улучшение деталей
         ↓
6. Бот отправляет итоговое фото с подписью:
   "✅ Готово! Фото улучшено: 4x апскейл + улучшение качества"
```

## Replicate модели

### Шаг 1 — Апскейл (Real-ESRGAN)
```python
output = replicate.run(
    "nightmareai/real-esrgan:42fed1c4974146d4d2414e2be2c5277c7fcf05fcc3a73abf41610695738c1d7b",
    input={
        "image": image_url,
        "scale": 4,
        "face_enhance": False  # лица улучшаем отдельно на шаге 2
    }
)
# output — URL результата, передаём в шаг 2
```

### Шаг 2 — Улучшение качества (clarity-upscaler)
```python
output = replicate.run(
    "philz1337x/clarity-upscaler:dfad41707589d68ecdccd1dfa600d55a208f9310748e44bfe35b4a6291453d5e",
    input={
        "image": output_from_step1,  # результат шага 1
        "scale_factor": 1,           # размер уже увеличен, только качество
        "sharpness": 1.5,
        "resemblance": 1.0,          # сохраняем оригинал
        "creativity": 0.15,          # минимальные изменения
        "output_format": "jpg"
    }
)
```

> Если clarity-upscaler недоступен — использовать
> `tencentarc/gfpgan:0fbacf7afc6817109e5a2a83d3a0a0e80cd4d85a6eb3ef9dc4c09d2e07b3ba8c`
> как запасной вариант для улучшения лиц и деталей.

## Обработка ошибок

| Ситуация | Поведение бота |
|---|---|
| Пользователь отправил не фото | «Пожалуйста, отправьте фото» |
| Фото больше 20 МБ (лимит Telegram) | «Файл слишком большой» |
| Replicate шаг 1 упал | До 3 повторных попыток с задержкой 2с, затем уведомить |
| Replicate шаг 2 упал | Отправить результат шага 1 с пометкой «частичная обработка» |
| Таймаут > 120 секунд | «Обработка заняла слишком много времени, попробуйте позже» |

## Важные детали реализации

- Скачивать фото из Telegram через `bot.get_file()` + `bot.download_file()`
  и загружать в Replicate как bytes или через временный URL
- Статусное сообщение (`⏳ Шаг 1/2...`) редактировать через `message.edit_text()`
  а не отправлять новое — чтобы не спамить
- Все вызовы Replicate делать асинхронно через `asyncio.to_thread(replicate.run, ...)`
  так как официальный SDK синхронный
- После отправки результата удалять временные файлы если они есть
